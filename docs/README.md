# docs

Documentation for this repo.

- [`spec/`](spec/) — dogfooded behavioral specs and conformance matrices for each workflow and skill. Each spec describes *what* and *why* (the contract); its sibling `<spec>.conformance.md` is a section-by-section audit (Conformed / Diverged / Not-built) of the spec against the actual implementation. All specs are `implemented`. Start at [`spec/README.md`](spec/README.md).
